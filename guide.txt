- config/	Contains configuration files, primarily for database connection settings and environment setup.

- controller/	Houses the business logic of the application. These functions handle requests from the routes, interact with the models, 
and format the response.

- middleware/	Stores middleware functions used to process requests before they hit the controller. Common uses include authentication, 
authorization, and logging.

- models/	Defines the data structure (schemas) for the database.

- Routes/	Defines the API endpoints (URLs) and maps them to the appropriate controller functions.

Node.js sequence

USER AUTHENTICATION/CREATION
1. /config   -> connect db or the database connection
2. /models   -> create model or the database schema
3. /middleware -> create middleware or the authentication
4. /controller -> create controller or the business logic
5. /routes -> create routes or the api endpoints
6. server.js -> create server or the main file

CREATION OF OTHER ENTITIES
1. /models -> create model or the database schema
2. /controller -> create controller or the business logic
3. /routes -> create routes or the api endpoints
4. server.js -> create server or the main file
